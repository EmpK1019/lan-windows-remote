# 局域网 Windows 远程控制 MVP

这是一个最小可运行版本：在被控制的 Windows 电脑上启动程序，局域网内另一台电脑、平板或手机用浏览器打开地址，输入控制端显示的一次性访问码，就可以查看屏幕并发送鼠标键盘操作。

它只适合控制你自己的电脑，或已经明确获得授权的电脑。程序不会隐藏运行、不会自启动、不会安装后台服务，也默认只接受本机和局域网地址访问。

## 运行

1. 在被控制的 Windows 电脑上安装 Python 3。
2. 双击 `start.bat`，或在这个文件夹中运行：

```powershell
python .\lan_remote.py
```

3. 窗口里会显示访问地址和访问码。
4. 在同一局域网的另一台设备浏览器中打开类似 `http://192.168.x.x:8765` 的地址。
5. 输入访问码后连接。

## 常用选项

只观看，不允许远程操作：

```powershell
python .\lan_remote.py --view-only
```

换端口：

```powershell
python .\lan_remote.py --port 9000
```

指定自己的访问码：

```powershell
python .\lan_remote.py --token "my-safe-code"
```

## 使用提示

- 控制端网页里点“键盘”后才会发送键盘按键；按 `Esc` 可以关闭键盘控制。
- 鼠标移动、点击和滚轮会直接发送到被控制电脑。
- 如果浏览器打不开地址，通常需要在 Windows 防火墙里允许 Python 在专用网络通信。
- 屏幕画面使用简单轮询和 Windows 自带 JPEG 编码，适合局域网临时控制；不是高帧率游戏串流。

## 当前限制

- 不能控制 Windows 锁屏界面或安全桌面里的 UAC 确认框。
- 普通权限运行时，可能无法操作管理员权限窗口；确实需要时可以用管理员身份启动。
- 复杂组合键、输入法候选窗、游戏全屏独占模式可能不完整。
- 多显示器会作为一张大画面显示。

## 安全边界

- 默认只允许本机、私有网段和链路本地地址访问。
- 每次启动都会生成新的访问码，除非你手动指定 `--token`。
- 关闭控制端窗口或按 `Ctrl+C` 就会停止服务。
- 不建议把它暴露到公网；如果需要跨互联网使用，请先加 TLS、账户体系、审计日志和更严格的授权确认。

## 后续可以继续做

- 打包成 `.exe`。
- 加系统托盘图标和“允许/拒绝本次连接”弹窗。
- 用 WebSocket 或视频编码降低延迟和流量。
- 加文件传输、剪贴板同步、会话日志。
