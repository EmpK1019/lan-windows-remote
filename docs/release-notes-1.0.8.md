# LAN Remote 1.0.8

- 修复 1.0.7 解锁成功后，已有控制窗口仍长期停留在锁屏末帧的问题。
- 安全桌面 HTTP 读取阻塞时，切源线程不再同步等待缓冲响应关闭；旧 socket 会立即失效，响应对象转入后台清理，Default 桌面编码器可以直接启动。
- 控制端检测到“锁定 → 已解锁”后，会严格等待原生视频 Stop 完成，再建立新的 H.264 连接并等待桌面关键帧，避免继续渲染 LockApp 旧帧。
- 修复原生视频子窗口隐藏等待首帧时不执行 D3D Present，进而形成“必须先渲染才显示、必须先显示才渲染”的首次连接与重连黑屏死锁；现在会在隐藏状态预渲染首帧，再无黑帧切换为可见。
- 新增真实 Windows 缓冲 HTTP 阻塞读取回归测试，以及静止安全源切回 Default 桌面的同连接测试。
- 新增打包后 WebView2 + 真实页面 + 原生 D3D 的锁屏解锁恢复 E2E，验证旧流中止、连接重建、Default 编码器启动及画面持续刷新。
- 本版本只修改锁屏解锁后的视频源恢复链路，不调整密码提交、鼠标键盘、帧率和画质策略。
